# ブランチを作成する

## 必要な情報の取得

ブランチ名にIssue番号を含める運用の場合、Issue番号が不明ならユーザーに確認する。
説明なしに数字だけが入力された場合は、それをIssue番号とみなして良い。
ブランチ名に変更内容を反映するため、必要なら SKILL.md「プロジェクト設定」のIssue管理場所からIssueの内容を取得する。

## 元ブランチ

特に指定がなければ、最新のmainブランチからブランチを作成する。
`git fetch origin main` を実行し、mainブランチに取り込まれていないリモートの変更があれば、ブランチ作成前に取り込む。

## ブランチ名

SKILL.md「プロジェクト設定」のブランチ命名規則に従う。
変更内容を表す部分は、長くなりすぎない範囲で内容に応じた名前をつける。
