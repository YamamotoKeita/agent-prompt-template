# CLAUDE.md の書き方

CLAUDE.md を更新する際は、必ずこのドキュメントの基準に従うこと。

## 基本原則

CLAUDE.md は毎セッション必ずコンテキストに読み込まれる。
短いほど1行あたりの重みが増し、長いほど記述が埋もれるため、なるべく短く保つ（目安は100行以内）

書く基準は次の2つだけ。

- **コードから導出できない事実**（コードを全部読んでも分からないこと）
- **世間のデフォルトと違う判断**（標準的なやり方が複数ある場面で、このプロジェクトが選んだこと）

## 書く場所の振り分け

AIエージェントへの指示は CLAUDE.md 以外でも指定できる。以下に従い適切なファイルに指示や手順を記載する。

| 内容 | 置き場所 |
|---|---|
| プロジェクト全体に効く制約・流儀 | CLAUDE.md |
| 特定のファイル種別やパスに紐づく制約・規約 | `.claude/rules/*.instructions.md` |
| 手順・やり方 | Agent Skills `.claude/skill/*/SKILL.md` |
| 深い参照知識（用語集・概要・環境情報など） | `docs/` に置き、CLAUDE.md からリンク |

**特定の種類のファイルを編集しているときだけ必要**なら rules、**設計・計画・レビューなど何をするときでも前提**になるなら CLAUDE.md。

## 「制約」セクションに書くこと

見た目やコードからは危険と判断できない、プロジェクト全体に効く禁止・必須事項のみ。

- 理由を必ず添える。理由があれば、書いていないケースにもモデルが類推を効かせられる
- 一般的な注意（テストを書く、秘密情報をコミットしない等）はモデルが知っているので書かない

## 流儀（必要になったら見出しを足す）

標準的なやり方が複数ある場面で、このプロジェクトが選んだ側とその理由。最初は無くてよい。モデルがデフォルトと違うことをやって直させた瞬間に、1個ずつ書き足していく。
